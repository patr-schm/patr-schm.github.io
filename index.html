I am a research assistant and PhD student at the Visual Computing Institute at RWTH Aachen University, Germany, where I work with Professor Leif Kobbelt.

My research interests are in computer graphics and geometry processing, with a focus on surface parametrization. I enjoy applying numerical optimization techniques to challenging real-world problems. Currently, I am working on algorithms that generate high-quality maps between different surfaces. Before that, I designed new algorithms for semi-automatic quad meshing (retopology).

Publication at SIGGRAPH Asia (2016)

Interactively Controlled Quad Remeshing of High Resolution 3D Models[1]

Publication C++ QT OpenMesh OpenFlipper

We propose an interactive retopology workflow for generating high quality quad meshes from unstructured triangle meshes. At every time, the user is provided with an automatically generated quad mesh, which can be modified by iteratively adding high-level constraints. To achieve updates at interactive speeds, we use a multiresolution approach introducing both a novel decimation scheme and a coarse-to-fine mapping method. I contributed key ideas to the mapping step and implemented large parts of the system.

[1] Hans-Christian Ebke, Patrick Schmidt, Marcel Campen, Leif Kobbelt. 2016. ACM Transactions on Graphics.

Master Thesis (2017)

Retrieval and Re-Embedding of Macro Constraints for Parametrization Based Quad Meshing

C++ Python Matlab OpenMesh OpenFlipper

In my Master Thesis, I developed a system that retrieves small surface patches from a database and embedds them on similarly curved regions of a target shape. Such patches carry user-defined information: e.g. feature curves to be used as constraints in interactive quad meshing scenarios. I came up with a new algorithm, computing a smooth map from a surface patch to a specified target region without tearing or foldovers.

Bachelor Thesis (2014)

Towards Interactive Quadrangulation through Local Remeshing

C++ OpenMesh OpenFlipper

I adapted the mixed-integer quadrangulation algorithm to work within local surface regions. This modification allows for faster updates and more controlled user interactions by limiting their effect to the desired area.

Personal Project (2019)

Software Renderer

Python NumPy Rendering Pipeline

I implemented an educational software renderer in Python.

It demonstrates the core computations that are essential to a forward rendering pipeline—with all performance related details stripped away. Using only NumPy as a library, it implements some key steps usually provided by an API such as OpenGL (e.g. rasterization, depth test and perspectively correct interpolation).
